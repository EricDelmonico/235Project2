Overall process was to first get the basic markup ready, then get the desired functionality in a working state, then make the page to look how we wanted it, then finally polish the javascript so that it was error-free.

Some features that aren’t necessarily apparent when looking at the page at face value are:

  • You can search by genre. This is done by typing nothing into the search bar, and having something selected in the Genre box
  • Special cases had to be set for this when sorting by rating. When sorting all shows in a genre, all results are considered, which means that the sorting method can be included in the URL used to access the API. When sorting shows that were searched for by text input from the user, only the most relevant (first-page) results should be considered. This meant a separate solution needed to be used to manually sort the results (this is reflected in the js).
  • Some functionality that is easy to re-implement, but was removed for jankiness, is the manually animated image that appears when a search is ongoing. It replaced the cursor with the animation, which is the reason we didn’t just use a gif. However, as said above, it was just a bit janky, so we moved it to a spot beside the search bar. (this is easier to see anyways)
